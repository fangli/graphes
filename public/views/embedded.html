Embedded GraphES enables the ability to integrate your website with awesome charts in few clicks.

1. Link to the following URL in your website

Href link

<a href="{{ baseLink }}/embedded/chart/?mini=false&k1=v1&k2=v2" target="_blank">See Chart</a>

2. or use embedded iframe in your page

Embedded iframe

{{ iframeContent[0] }}mini=true{{ iframeContent[1] }}

3. or get RAW data from data service

HTTP POST

POST TO: {{ baseLink }}/api/es/facets

Data

{
  "index_pattern": "",
  "time_field": "",
  "value_field": "",
  "interval": "",
  "from": "",
  "to": "",
  "series": {
    "demoSeries1": [
      "project:\"test\"",
      "release:\"test\"",
      "env:\"test\""
    ],
  }
  "token": "", //token is used for authentication
}

Data Params

Name Options Description
index_pattern * String The ES indice pattern
time_field String. Default to "@timestamp" The field name which contains metric timestamp
value_field String. Default to "_value" The field name which contains value
interval Time with unit, such as 30s, 5m, 1d. Default to "5m" The interval between each data points
from UNIX Timestamp(in Milliseconds). Default to "24 hours ago" Start time for series
to UNIX Timestamp(in Milliseconds). Default to "now" End time for series

Series Params *

You could have multiple data series in the same chart, with different queries and filters, using params array series.

Parameter Array series must contains a series name like "series.mySeries1", duplicated name will be considered as relationship "AND".

For example

    series.mySeries1 = project:"projectA" & series.mySeries1 = release:"releaseB" & series.mySeries1 = env:"production" &

    series.mySeries2 = project:"projectB" & series.mySeries2 = release:"releaseB" & series.mySeries2 = env:"development"

These series params will give you two individual series in the same chart, mySeries1 and mySeries2. Use URL encoding if the queries contain special words.


Style Params

Name Options Description
title String The main title of chart
yaxis String The Y-Axis title of chart
style_type area | line | column The type of chart, should be one of area, line or column
style_value min | mean | max | count | total Which data series will be shown
style_stacking (empty) | normal | percent Leave this param empty if you want to disable it
